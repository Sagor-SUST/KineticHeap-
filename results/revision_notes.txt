=== R10: Trajectory Update API ===
Add to Section IV (The KineticHeap++ Data Structure), new subsection E:
"E. Trajectory Update API
When element i undergoes a slope or intercept change (ai, bi) -> 
(ai', bi') at time t, the update proceeds as:
1. Apply lazy offset: ai <- ai + bi*(t - t_last_i), t_last_i <- t
2. Set new trajectory: ai <- ai', bi <- bi'  
3. For each heap-adjacent pair (i,j): delete old certificate, 
   compute new t_ij, insert if t_ij > t.
This restores I1-I3 in O(log n) amortised time. Applications such as 
moving-object databases or financial streams with frequently changing 
slopes should batch updates where possible to amortise certificate 
re-insertion costs."
